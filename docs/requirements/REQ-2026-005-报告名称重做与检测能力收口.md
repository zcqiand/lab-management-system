## 1. 需求描述

### 用户原话（逐字照抄）

> 报告模板改成报告名称，报告名称放到检测能力下面，检测项目与报告名称多对多关系，报告名称需要关联检测项目，关联检测标准（区分检测依据和判定依据），关联检测参数，将型号、规格、等级、牌号、报告名称拆分到types，每个类都一个文件，原来基础数据下面报告类别，合同类别，标准管理，检测参数不要了

### 理解后的需求描述

把"报告模板"这一旧概念彻底替换为新的"报告名称"实体。报告名称是检测能力（M06）下的中转型实体，按 `code + name` 标识，挂载四类多对多关联：

| 关联 | 中间表 | 备注 |
|---|---|---|
| 检测项目 ↔ 报告名称 | `InspectionObjectReportName` | 新增 |
| 报告名称 ↔ 检测标准 | `InspectionReportNameStandard` | 含 `role: TESTING \| JUDGMENT` 字段，复用 `InspectionStandardRole` 枚举 |
| 报告名称 ↔ 检测参数 | `InspectionReportNameParameter` | 新增 |

报告名称菜单从原 M04 基础数据下，移到 M06 检测能力下。原 M04.F02 报告模板（含 HTML 模板 + `{{标签}}` 字段）整体丢弃，不留回退。

M04 基础数据下四个功能整组废弃：报告类别（M04.F01）、报告模板（M04.F02，整组）、标准管理（M04.F03）、检测参数（M04.F04）、合同类别（M04.F10）。这些功能标记状态改为"已废弃"，不再允许新增/编辑/删除，**保留旧数据不物理删除**（被接样单等历史数据引用）。

型号/规格/等级/牌号（M04.F06/F07/F08/F09）从「InspectionTechnicalRequirement 上的字符串字段」提为独立实体（`InspectionBrand` / `InspectionModel` / `InspectionGrade` / `InspectionSpec`），各占一个 type 文件，CRUD 页面继续保留在 M04 下，但过滤维度从"报告类别"改为"检测专项"（报告类别已废弃）。`InspectionTechnicalRequirement` 上的 brand/model/grade/spec 字段相应改为 FK 引用这四个新实体。

类型文件层面，新增 5 个：`inspection-report-name.ts`、`inspection-brand.ts`、`inspection-model.ts`、`inspection-grade.ts`、`inspection-spec.ts`。复用现有 `inspection-standard-role.ts`。

报告 HTML 的渲染路径不在本 REQ 范围，后续单独 REQ。

## 5. 流程影响

```text
检测专项 (M06.F01)
  ↓
型号 / 规格 / 等级 / 牌号 (M04.F06-F09，按专项过滤)
  ↓
检测项目 (M06.F02) ── 检测参数 (M06.F03) ── 检测标准 (M06.F04, 含 role=TESTING/JUDGMENT)
  ↓
计算规则 (M06.F05) → 技术要求 (M06.F06，按项目+参数+判定依据+条件)
  ↓
报告名称 (M06.F07) —— 关联 检测项目 / 检测标准(TESTING) / 检测标准(JUDGMENT) / 检测参数
  ↓
（不在本 REQ 范围）HTML 报告渲染
```

新增流程节点：报告名称作为"按场景组合检测资源"的容器，向上为后续报告生成提供组合配置，向下汇聚检测能力四类实体的多对多关联。

## 6. 风险与回滚

| 风险 | 影响面 | 缓解 | 回滚方式 |
|---|---|---|---|
| 报告名称取代报告模板，旧接样单中 reportTemplateId 字段无对应新实体 | 报告生成 | 旧字段保留兼容，新功能走 reportNameCode | 关闭 M06.F07，旧字段重新启用 |
| M04.F01-F04/F10 整组废弃，被接样单等历史数据引用 | 历史数据读取 | 旧 API 读路径保留供查询，写入路径返回 410 Gone | 重新暴露旧菜单入口（状态改回已上线） |
| 型号/规格/等级/牌号提为实体，原 TechnicalRequirement 中是字符串字段，迁移期数据可能匹配失败 | 单项评定 | 迁移期双轨：TechnicalRequirement 字段保留原字符串，新增 *Code FK 字段；评定逻辑优先 FK，回退字符串 | 回退字段类型为 string，删除新实体 |
| 新增 5 个 type 文件与既有 13 个 type 文件的 barrel 导出顺序冲突 | 类型/构建 | barrel 顺序追加，不重排既有 | 删除新增 5 个 barrel 导出 |
| 删除保护（FK 引用）触发后旧数据无主 | UI/数据 | 删除前提示被引用数量，提供"先解绑再删除"流程 | 解除保护直接删除 |
| role enum 复用 vs 新 enum 的不一致 | 后续扩展 | 严格使用 InspectionStandardRole，不引入新枚举值 | 不适用 |
