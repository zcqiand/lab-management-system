## 1. 需求描述

### 用户原话

lab-management-system 增加合同信息字段：合同类别、建设单位、监理单位、送检人员姓名、送检人员联系电话

### 理解后的需求描述

在现有合同（Contract）实体上扩展 5 个字段：

| 需求字段 | 说明 | 现状 |
|---|---|---|
| 合同类别 | 合同分类，需形成码表支持 CRUD | **不存在**，需新增 M04.F10 合同类别码表管理 |
| 建设单位 | 项目的建设单位 | **不存在**，直接新增字段到合同表单 |
| 监理单位 | 项目的监理单位 | **不存在**，直接新增字段到合同表单 |
| 送检人员姓名 | 送检人员姓名（与见证人不同） | **不存在**，直接新增字段到合同表单 |
| 送检人员联系电话 | 送检人员联系电话（与见证人电话不同） | **不存在**，直接新增字段到合同表单 |

**注意**：代码中已有的 `constructionUnit`（施工单位）、`witnessUnit`（见证单位）、`witness`（见证人）、`witnessPhone`（见证人电话）与需求字段**语义不同**，需分别保留，新增的 4 个字段独立存在。

现有 `Contract` 接口（需扩展）：

```typescript
export interface Contract {
  id: string
  contractCode: string
  clientUnit: string        // 委托单位
  projectName: string       // 项目名称
  projectLocation?: string  // 项目地点
  // 现有字段（保留）
  constructionUnit: string  // 施工单位
  witnessUnit: string       // 见证单位
  witness: string           // 见证人
  witnessPhone?: string     // 见证人电话
  contactPerson?: string    // 联系人
  contactPhone?: string     // 联系电话
  entrustedDate?: string    // 委托日期
  status: ContractStatus
  createdAt: string
  updatedAt: string
  // 待新增字段
  contractCategory?: string      // 合同类别（关联 M04.F10）
  buildingUnit?: string          // 建设单位
  supervisionUnit?: string       // 监理单位
  inspectionPerson?: string      // 送检人员姓名
  inspectionPhone?: string       // 送检人员联系电话
}
```

### 澄清记录

| 疑问 | 澄清结论 | 澄清人 | 日期 |
|---|---|---|---|
| 「合同类别」是什么类型？预定义枚举还是自由文本？ | 形成码表，可对合同类别进行增删改查（CRUD） | 用户 | 2026/07/11 |
| 「送检人员」与现有「见证人」是否为同一角色？ | 不是同一角色，需分别保留 | 用户 | 2026/07/11 |
| 「建设单位」和「监理单位」是否已有对应单位管理功能？ | 无需单独管理页面，直接在合同表单中维护字段 | 用户 | 2026/07/11 |
