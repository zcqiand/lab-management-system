# lab-management-system — 功能规格文档

> 本文档描述 lab-management-system 的所有功能特性、验收标准与测试策略。
> 与 [CLAUDE.md](CLAUDE.md) 共同构成本仓的完整规范文档。

## 功能清单

### 认证与权限

| 功能名称 | 功能描述 | 验收标准 |
| :--- | :--- | :--- |
| JWT 登录 | 用户名+密码登录，MSW mock 层签发 JWT token | 登录成功返回 token；无效凭证返回 401 |
| RBAC 角色权限 | admin（全部权限）/ technician（受限权限）两级角色 | admin 可删除用户；technician 的 user:delete 菜单项不渲染 |
| 路由守卫 ProtectedRoute | 未登录跳转登录页；角色不匹配跳转 403 | 三态（未登录/已登录/角色不匹配）均正确拦截 |
| 权限指令 hasPermission | 按权限码动态渲染/卸载 UI 元素 | 无权限元素从 DOM 移除，不只是隐藏 |

### 接样管理

| 功能名称 | 功能描述 | 验收标准 |
| :--- | :--- | :--- |
| 接样单 CRUD | 创建/查看/编辑/删除接样单，含 categoryCode 报告类别 | 列表加载 10 条种子数据；创建后立即出现在列表 |
| 报告类别关联 | 接样单关联报告类别（steel/cement/concrete/sand/gravel/rebar_mech/rebar_weld） | 新建样品时按接样单的 categoryCode 过滤四码表选项 |
| 流程状态 | 接样单含 flowStatus（待提交/审核中/已批准/已发放/已归档）+ flowHistory | 状态流转后 history 追加记录，含操作人/时间/动作 |

### 样品管理

| 功能名称 | 功能描述 | 验收标准 |
| :--- | :--- | :--- |
| 样品 CRUD | 样品归属接样单，含 receiptId/型号/规格/等级/牌号 | 新建样品后自动出现在接样单详情页 |
| 四码表组合框 | 型号/规格/等级/牌号均为 input+datalist，可输入可选择 | 按报告类别过滤选项；允许自由输入 |
| 扩展属性渲染 | 按报告类别动态渲染 extFields（如钢材的炉号、混凝土的浇筑部位） | 新建样品时仅显示当前类别定义的扩展字段 |

### 检测项与自动评定

| 功能名称 | 功能描述 | 验收标准 |
| :--- | :--- | :--- |
| 检测项 CRUD | 样品下创建检测项（parameter + value + passed） | 单项检测记录归属 sampleId |
| 自动评定 | 录入检测值后按「技术要求」码表自动评定合格/不合格 | evaluateTestResult 按牌号+4/型号+2/等级+2/规格+1 打分取最高；无法匹配时 autoPassed=null |
| 人工改判 | 无法自动评定的项可手工改判 passed | PUT /test-items/:id 显式传 passed 覆盖自动结果 |

### 报告文档

| 功能名称 | 功能描述 | 验收标准 |
| :--- | :--- | :--- |
| 报告模板维护 | 每个报告类别一份 HTML 模板，{{标签}} 插入点 | 模板列表显示所有 7 个类别的模板；可新增/编辑/删除 |
| 报告预览 | 按模板合成机构信息/合同/接样单/样品表/检测结果表 | 预览模态框显示完整报告，含 samplesTable（含扩展属性列）和 testItemsTable |
| Word 下载 | 生成 .doc 文件供下载 | 点击下载触发 docx 生成，含所有报告数据 |
| 报告查看入口 | 审核/批准/发放/归档四个阶段页均提供「查看报告」按钮 | 任意阶段均可打开对应接样单的已生成报告 |

### 流程引擎

| 功能名称 | 功能描述 | 验收标准 |
| :--- | :--- | :--- |
| 单一流程线 | 接样→任务安排→数据录入→报告审核→报告批准→报告发放→报告归档 | 全链路端到端测试通过 |
| 批量提交 | 选中多条接样单批量提交 | POST /api/receipts/flow {action:'submit',ids,operator} 批量推进状态 |
| 批量退回 | 选中多条接样单批量退回 | action:'return' 批量回退至上一环节 |
| 撤回 | 提交人主动撤回（仅当提交后未被下一环节处理时） | action:'withdraw' 撤回当前用户的提交记录 |
| 通用阶段页面 | FlowStagePage 统一渲染各阶段列表+批量操作+「我提交的（可撤回）」区块 | 所有阶段页路由正常；权限按钮正确显示 |

### 统计汇总

| 功能名称 | 功能描述 | 验收标准 |
| :--- | :--- | :--- |
| 报告汇总表 | 按报告类别输出试验报告汇总表（钢材/水泥/砂/碎石/混凝土/机械连接/焊接连接） | 三种 summaryType（material/concrete/connection）列定义各异 |
| 工程过滤 | 按合同（工程）过滤汇总数据 | 只统计已生成报告编号的接样单 |
| 统计计算 | computeStats 按样品维度聚合 | 统计值与种子数据一致 |

### 基础数据管理

| 功能名称 | 功能描述 | 验收标准 |
| :--- | :--- | :--- |
| 机构信息管理 | 平台级机构信息维护 | 路由 /organization 可访问 |
| 角色管理 | 定义角色名称/描述/权限列表 | 角色列表含 admin/technician 两个内置角色 |
| 用户管理 | 用户 CRUD（含密码修改） | admin 可删除用户；technician 无 user:delete 权限 |
| 报告类别 CRUD | 新增/编辑/删除报告类别（含 extFields 扩展属性配置） | 被接样单引用的类别不可删除；code 不可修改 |
| 报告类别标准关联 | 报告类别与检测标准的多对多关联 | CategoryStandardList 正确维护关联 |
| 参数/标准/技术要求 | TestParameter / TestStandard / TechnicalRequirement 码表维护 | TechnicalRequirement 按四维度匹配样品 |
| 报告模板 | ReportTemplateList 含新增/编辑/删除 | 每个类别有唯一模板 |

### 码表管理

| 功能名称 | 功能描述 | 验收标准 |
| :--- | :--- | :--- |
| 型号管理 | /models，按报告类别过滤 | 新建样品时仅显示当前类别已维护的型号 |
| 规格管理 | /specifications | 同上 |
| 等级管理 | /grades | 同上 |
| 牌号管理 | /brands | 同上 |
| CategoryDictList | 四码表通用页，统一 input+datalist 交互 | 按 categoryCode 过滤选项 |
